# AI Usage Document: Tool Interactions & Core Corrections

This document highlights the tool interactions during the project construction and records the major corrections applied to generated code blocks to meet correctness standards.

---

### 1. Developer Tool & Subagent Logs
* **Grep Searches**: Used to quickly trace references of `cors`, `VITE_API_URL`, and `fuzzyMatchUser` through the frontend and backend files.
* **Direct File Views**: Utilized to read full contents of `importService.js`, `splitCalculator.js`, `vite.config.js`, and `axios.js` to ensure edits were contiguous and syntactically clean.
* **Terminal Invocation**: Used `npm run build` and `npm start` to execute the code and identify runtime issues early.

---

### 2. Major Corrections Applied to AI Code

The following sections document four key instances where code generated by the AI was incorrect or failed to meet requirements, and details how they were corrected.

#### Correction 1: Express 5 Catch-All Wildcard Route Crash
* **Incorrect Generated Code**:
  ```javascript
  app.get('*', (req, res) => {
    res.sendFile(path.join(__dirname, '../public/index.html'));
  });
  ```
* **Failure Mode**: The backend server crashed immediately on startup, throwing:
  `PathError [TypeError]: Missing parameter name at index 1: *`
  Express 5.x uses path-to-regexp v8 which no longer allows standalone, unnamed asterisks.
* **Correction**: Changed the wildcard route to use a named splat parameter:
  ```javascript
  app.get('/*splat', (req, res) => {
    if (!req.path.startsWith('/api')) {
      res.sendFile(path.join(__dirname, '../public', 'index.html'));
    }
  });
  ```

#### Correction 2: Strict Sum Validation for Percentage Splits
* **Incorrect Generated Code**:
  ```javascript
  const totalPct = splitDetails.reduce((sum, d) => sum + d.percentage, 0);
  if (totalPct !== 100) {
    throw new Error("Percentage splits must sum to 100%");
  }
  ```
* **Failure Mode**: Failed for standard floating-point splits (e.g. 33.33% + 33.33% + 33.34% summed to 100.00000000000001, which triggered the strict inequality check and crashed).
* **Correction**: Updated validator in `splitCalculator.js` to use an epsilon range check:
  ```javascript
  if (Math.abs(totalPct - 100) > 0.02) {
    throw new Error(`Percentage splits must sum to 100%, got ${totalPct}%`);
  }
  ```

#### Correction 3: Missing Time-Bounded Date Filters on User Balances
* **Incorrect Generated Code**: The initial balance summarizer computed net sheets by aggregating all history regardless of dates:
  ```javascript
  const net = paidAmount - owedAmount;
  ```
* **Failure Mode**: Charged Sam (who joined mid-April 2026) for the February and March rent sheets.
* **Correction**: Added a membership filter checking date bounds:
  ```javascript
  // Only include an expense if the user was active on the expense date
  const activeOnDate = joinedAt <= date && (leftAt === null || leftAt >= date);
  ```

#### Correction 4: Local Storage Authentication Storage Choice
* **Incorrect Generated Code**: Suggested saving both JWT access tokens and refresh tokens in HTML5 `localStorage`.
* **Failure Mode**: Left the session tokens vulnerable to token theft via Cross-Site Scripting (XSS).
* **Correction**: Refactored token system to serve JWT Refresh Tokens through HTTP-Only cookies and keep access tokens in volatile React application memory.
